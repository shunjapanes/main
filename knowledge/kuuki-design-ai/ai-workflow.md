## 「爆速AIワークフロー」の本質

シリコンバレー流のAI活用は「ツールを使う」ではなく「ワークフローを設計する」。

**キーポイント**: AIに仕事をさせるのではなく、AIが自律的に動ける「仕組み」を作る。

---

## プロのAI使い方：実演から学べること

### 旧来の使い方（初心者）

```
人間 → プロンプト入力 → AI出力 → 人間が修正 → 完成
```

### シリコンバレー流（プロ）

```
人間が設計した文脈 → AI呼び出し → AI出力 → 自動検証 → 次のAIステップへ
         ↑                                              ↓
         └──────────── 設計済みフィードバックループ ──────┘
```

---

## ワークフロー設計の原則

### 1. タスクを「AIが扱える粒度」に分解する

- 大きなタスクをそのまま投げない
- AIが一度のターンで完結できる単位に分割する
- 依存関係を意識して順序を設計する

### 2. 出力フォーマットを先に設計する

- AIに出力形式を決めさせない
- JSON・Markdown・箇条書きなど形式を明示する
- 後続のステップで使いやすい形式を逆算して指定する

### 3. 検証ステップをワークフローに組み込む

- AIの出力を人間がゼロから見直さない
- 「このアウトプットが正しいか確認せよ」というAIへの指示も設計する
- セルフチェック機能をプロンプトに組み込む

### 4. 人間はジャッジに集中する

- 実行はAIに任せる
- 人間がすべきことは「判断」「方向性の修正」「優先順位付け」
- 手を動かす作業からジャッジ作業にシフトする

---

## 実際の活用例（デザインプロセス）

| フェーズ | AIへの委任内容 | 人間の役割 |
|---|---|---|
| リサーチ | 競合分析・情報収集・まとめ | 方向性の確認 |
| アイデア出し | 複数の選択肢を生成 | 選択・絞り込み |
| プロトタイプ | ワイヤーフレーム説明・コード生成 | レビュー・修正指示 |
| ドキュメント | 仕様書・説明文の草稿作成 | 精度チェック・承認 |
| QA | テストケース生成・バグ候補の洗い出し | 優先度判断 |

---

## ツールの選び方

- 特定のAIツールに縛られない（文脈エンジンがあればどのAIでも使える）
- 目的に応じてClaude / ChatGPT / Gemini を使い分ける
- 大事なのはツールではなくワークフローの設計品質
