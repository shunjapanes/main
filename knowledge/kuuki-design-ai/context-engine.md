# 文脈エンジン（Context Engine）

> ソース: クウキデザイン YouTube / [kuukidesign.studio](https://www.kuukidesign.studio/)

---

## 概念

**文脈エンジン** = どのAIでも「説明ゼロ」で動かすための設計と実装。

毎回「私はこういう人間です」「このプロジェクトはこういう背景です」と説明する必要をなくすシステム。
シリコンバレーで実際に毎日稼働させているObsidianベースの仕組みを公開したもの。

---

## なぜ必要か

### AIプラットフォームの「文脈の囲い込み」問題

- ChatGPT、Claude、Gemini など各AIは独自のコンテキスト管理を持つ
- プラットフォームをまたぐと文脈がリセットされる
- プロンプトを毎回磨いても根本解決にならない

**本質的な問題**: プロンプトではなく「文脈の構造設計」が不足している。

---

## 文脈エンジンの構造（Obsidianベース）

### 設計思想から実装まで全公開

```
文脈エンジン
├── 自己文脈ファイル（Who I am）
│   - 役割・専門性・思考スタイル
│   - よく使うツール・プロジェクト背景
│
├── プロジェクト文脈ファイル（What this is）
│   - 各プロジェクトの背景・目標・制約
│   - ステークホルダー情報
│
├── タスク文脈ファイル（What to do）
│   - 今やるべきことの文脈
│   - 依存関係・優先度
│
└── AI呼び出しテンプレート
    - 文脈を自動で組み立ててAIに渡す
```

---

## 実装のポイント

1. **Obsidianを中央ハブにする** — ノートシステムとAI指示系統を統合
2. **文脈ファイルを事前に設計** — 人間が書いた文脈をAIが読める形式で保存
3. **テンプレートで文脈を自動挿入** — AIを呼ぶたびに手動で説明する手間をゼロに
4. **どのAIでも同じ文脈から始められる** — Claude でも ChatGPT でも同じ精度を担保

---

## 応用

- Claude Projects や ChatGPT GPTs のシステムプロンプトに文脈エンジンの出力を貼る
- Obsidianのデイリーノートと連携してその日の文脈を自動生成
- プロジェクトごとに文脈ファイルを切り替えてAIの振る舞いを変える
