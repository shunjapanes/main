## Claudeの特徴（2026年6月時点）

| モデル | モデルID | 特徴 |
|---|---|---|
| Claude Opus 4.8 | `claude-opus-4-8` | 最も高性能。長時間の自律エージェント作業・知的作業に強い。1Mコンテキスト |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | 速度と知能のバランス。1Mコンテキスト |
| Claude Haiku 4.5 | `claude-haiku-4-5` | 最速・低コスト。シンプルなタスク向け |

- **大きなコンテキストウィンドウ**（Opus/Sonnetは1M）→ 文脈エンジンと相性が良い
- **Claude Code**（CLI/IDE）でエージェント的にコードを書ける
- **Projects**（claude.ai）でプロジェクト単位の文脈を保持
- **Skills** で再利用可能な専門知識をオンデマンド読み込み

## 文脈エンジンをClaudeに実装する

クウキデザインの[文脈エンジン](../context-engine.md)はClaudeで特に有効。大きなコンテキストを活かせる。

### 1. Claude Projects に文脈を載せる

- Projectのカスタム指示（システムプロンプト）に「自己文脈」を貼る
  - 役割・専門性・思考スタイル・よく使うツール
- Project Knowledge にプロジェクト背景ドキュメントをアップロード
- → 毎回説明ゼロで、そのプロジェクトの文脈から会話が始まる

### 2. Claude Code には `CLAUDE.md` を置く

```
プロジェクトルート/
└── CLAUDE.md   ← 文脈エンジンの出力をここに
```

- プロジェクト概要・規約・運用ルール・注意事項を記載
- Claude Codeが自動で読み込み、セッションをまたいで同じ文脈から始められる
- まさにObsidianベース文脈エンジンの「CLAUDE.md版」

### 3. Obsidian文脈エンジン → Claude へのブリッジ

- Obsidianの自己文脈・プロジェクト文脈ファイルをコピペ、またはMCP経由で接続
- ClaudeのMCP対応により、Obsidian vaultを直接読ませることも可能

## 7原則のClaude適用

[ai-mindset.md の7原則](../ai-mindset.md) をClaude向けに具体化:

| 原則 | Claudeでの実践 |
|---|---|
| ゴールを先に設定 | 会話冒頭で達成したい成果物を明示 |
| 条件を全て先に出す | 制約・前提を最初のメッセージにまとめる（後出ししない） |
| 文字数を指定 | 「300字で」「箇条書き5項目で」など出力長を明示 |
| トーンを決める | 「丁寧に」「カジュアルに」「技術文書として」 |
| 一度に詰め込みすぎない | 大タスクは分割。Claude Codeなら計画→実行を分ける |
| 段階的に進める | Opus 4.8は長時間タスクが得意 → 最初の1ターンで全体仕様を渡すと効率的 |
| 選択肢を比較させる | 「3案出して、それぞれの長所短所を比較して」 |

## ワークフロー設計（Claude Code）

[ai-workflow.md](../ai-workflow.md) のシリコンバレー流をClaude Codeで:

```
人間が設計した文脈(CLAUDE.md) → Claude Code起動
  → 計画立案 → 実装 → テスト実行(自動検証) → 次ステップ
         ↑                                      ↓
         └────── 設計済みフィードバックループ ───┘
```

- **タスク分解**: Claudeに「まず計画を立てて」と指示し、人間が承認してから実行
- **出力フォーマット指定**: コード規約・ファイル構成を先に決める
- **検証の組み込み**: 「変更後にテストを走らせて結果を確認して」
- **人間はジャッジ**: Claudeが手を動かし、人間は方向性と承認に集中

## Claude固有のTips

- **長時間タスクはOpus 4.8 + 高effort**: 最初の1ターンに全仕様を渡す（クウキデザインの「条件先出し」と一致）
- **Skills**: 繰り返す専門タスクはSkill化してオンデマンド読み込み
- **MCP**: 外部ツール（GitHub・Slack・DB）を文脈エンジンに接続
- **プロンプトより文脈設計**: Claudeは指示に忠実 → 文脈を構造化して渡すのが効く
