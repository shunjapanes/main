# Gemini — クウキデザイン流の使い方

> クウキデザインの「文脈エンジン」「7原則」「ワークフロー設計」をGeminiに当てはめた実践ガイド。
> 関連: [../ai-mindset.md](../ai-mindset.md) / [../context-engine.md](../context-engine.md) / [../ai-workflow.md](../ai-workflow.md)

---

## Geminiの特徴

- **巨大なコンテキストウィンドウ**: 長大なドキュメント・コードベースをまとめて読ませられる
- **Google Workspace連携**: Gmail・Docs・Drive・Sheetsと統合
- **Gems**: 特定用途のカスタムGemini（ChatGPTのGPTs相当）
- **Saved Info**: ユーザーの文脈を記憶
- **マルチモーダル**: 画像・動画・音声の理解に強い

---

## 文脈エンジンをGeminiに実装する

クウキデザインの[文脈エンジン](../context-engine.md)をGeminiで:

### 1. Saved Info / Gems に自己文脈を載せる

- Saved Info に役割・専門性・思考スタイルを記載
- Gems のカスタム指示にプロジェクト文脈を固める
- → 毎回の説明を省く（文脈エンジンの中核）

### 2. 巨大コンテキストを活かす

- Geminiの長大なコンテキストウィンドウは文脈エンジンと好相性
- Obsidianの文脈ファイル群・プロジェクト資料をまるごと貼り付けられる
- 「文脈の囲い込み」をコンテキスト量で乗り越える

### 3. Workspace連携で文脈を自動供給

- Drive内のプロジェクト資料をGeminiに参照させる
- Docs/Sheetsの内容を文脈として直接読ませる

## 7原則のGemini適用

[ai-mindset.md の7原則](../ai-mindset.md):

| 原則 | Geminiでの実践 |
|---|---|
| ゴールを先に設定 | 冒頭で成果物を定義 |
| 条件を全て先に出す | 制約・前提をまとめて渡す |
| 文字数を指定 | 出力長を明示 |
| トーンを決める | Saved Info / Gemで既定トーンを設定 |
| 一度に詰め込みすぎない | タスクを分割 |
| 段階的に進める | ステップごとに確認を挟む |
| 選択肢を比較させる | 「複数案を比較表で」 |

## ワークフロー設計

[ai-workflow.md](../ai-workflow.md) のシリコンバレー流:

- **タスク分解**: AIが扱える粒度に分ける
- **出力フォーマット指定**: 形式を逆算して指定
- **検証ステップ**: セルフチェックをプロンプトに組み込む
- **人間はジャッジ**: 実行・収集はGeminiに任せ、判断に集中

## Gemini固有のTips

- **長文脈が最大の武器**: 文脈エンジンのファイル群をまとめて読ませる
- **Workspace連携で文脈を自動化**: Drive/Docsを文脈ソースに
- **マルチモーダル文脈**: 画像・動画も文脈の一部として渡せる
- **Gems = プロジェクト文脈の固定化**: 繰り返す用途はGem化
