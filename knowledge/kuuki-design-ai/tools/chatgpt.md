## ChatGPTの特徴

- **Custom Instructions**: 全会話に共通する文脈を設定できる
- **GPTs**: 特定用途のカスタムChatGPTを作れる（システムプロンプト+知識ファイル）
- **Memory**: 会話をまたいでユーザー情報を記憶
- **Projects**: 関連する会話と文脈をまとめる
- **Code Interpreter / Canvas**: コード実行・文書/コード編集

---

## 文脈エンジンをChatGPTに実装する

クウキデザインの[文脈エンジン](../context-engine.md)をChatGPTで:

### 1. Custom Instructions に自己文脈を載せる

- 「あなたについて」欄に役割・専門性・思考スタイル・よく使うツールを記載
- 「どう応答してほしいか」欄にトーン・出力フォーマットの好みを記載
- → 毎回自己紹介する必要がなくなる（文脈エンジンの中核）

### 2. GPTs でプロジェクト文脈を固める

- プロジェクトごとにGPTを作成
- システムプロンプトに「プロジェクト文脈」（背景・目標・制約）を記載
- Knowledge にプロジェクト資料をアップロード
- → ChatGPTのProjects機能や文脈の囲い込みに対するクウキデザインの解決策と一致

### 3. Obsidian文脈エンジン → ChatGPT

- Obsidianの文脈ファイルをCustom InstructionsやGPTのシステムプロンプトに貼る
- 「文脈の囲い込み」を超えて、同じ文脈をChatGPTでも再現

## 7原則のChatGPT適用

[ai-mindset.md の7原則](../ai-mindset.md):

| 原則 | ChatGPTでの実践 |
|---|---|
| ゴールを先に設定 | 最初のメッセージで成果物を定義 |
| 条件を全て先に出す | 前提・制約をまとめて渡す |
| 文字数を指定 | 「500字で」「3段落で」 |
| トーンを決める | Custom Instructionsで既定トーンを設定 |
| 一度に詰め込みすぎない | タスクを分割して順に依頼 |
| 段階的に進める | 「ステップ1から始めて、私の確認を待って」 |
| 選択肢を比較させる | 「3つの案を表で比較して」 |

## ワークフロー設計

[ai-workflow.md](../ai-workflow.md) のシリコンバレー流:

- **タスク分解**: 大きな依頼を扱える粒度に分ける
- **出力フォーマット指定**: JSON・Markdown・表など形式を逆算して指定
- **検証ステップ**: 「この出力が要件を満たすか確認して」をプロンプトに含める
- **人間はジャッジ**: Code InterpreterやCanvasで実行はAIに任せ、判断に集中

---

## ChatGPT固有のTips

- **Custom Instructions = 簡易文脈エンジン**: まずここを固めるのが最優先
- **GPTs = プロジェクト文脈の固定化**: 繰り返す用途はGPT化
- **Memoryは補助**: 自動記憶に頼りすぎず、重要な文脈は明示的に設計する
- **プロンプトの磨きすぎに注意**: クウキデザインの指摘どおり、プロンプト改善より文脈構造の設計が効く
