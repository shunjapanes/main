# Claude Code — クウキデザイン流の使い方

> クウキデザインの「AIワークフロー」「AIRFLOW」思想をClaude Code（Anthropic公式CLI/IDEエージェント）に当てはめた実践ガイド。
> 関連: [../ai-workflow.md](../ai-workflow.md) / [../airflow-app.md](../airflow-app.md) / [claude.md](./claude.md)

---

## Claude Codeとは

Anthropic公式のエージェント型コーディングツール。ターミナル・デスクトップアプリ・Web（claude.ai/code）・IDE拡張（VS Code / JetBrains）で動作。
人間とAIが協業してコードを書く——まさにクウキデザインの「AIRFLOW」思想（人間とAIが同じボードを読み書きする）の実装系。

---

## AIRFLOW思想との対応

[airflow-app.md](../airflow-app.md) の「担当を割り当てたら後はAIが勝手に動く」をClaude Codeで:

| AIRFLOWの考え | Claude Codeでの実現 |
|---|---|
| 人間とAIが同じボードを見る | リポジトリ・差分・テスト結果を共有 |
| AIへの委任を設計する | タスクを伝え、Claude Codeが計画→実装→検証 |
| 人間の判断をどこに置くか | 計画承認・コードレビュー・マージ判断 |
| AIの出力を次に接続 | テスト→修正→PRの自動連鎖 |

## 文脈エンジンとしての CLAUDE.md

[context-engine.md](../context-engine.md) の「説明ゼロで動かす」をClaude Codeで実現する中核ファイル。

```
プロジェクトルート/
└── CLAUDE.md   ← 文脈エンジンの出力
```

記載する内容:
- プロジェクト概要・技術スタック
- コーディング規約・命名規則
- 運用ルール・注意事項
- よく使うコマンド（テスト・ビルド・lint）

→ セッションをまたいで毎回同じ文脈から会話を始められる。Obsidianベース文脈エンジンの「Claude Code版」。

## ワークフロー設計（シリコンバレー流の実装）

[ai-workflow.md](../ai-workflow.md) のループをClaude Codeで:

```
CLAUDE.md(文脈) → Claude Code起動
  → 計画立案 → 人間が承認 → 実装
  → テスト自動実行(検証) → 結果確認 → 次ステップ
         ↑                                  ↓
         └──── 設計済みフィードバックループ ──┘
```

### 実践ポイント

1. **計画と実行を分ける** — 「まず計画を立てて」と指示し、承認してから実装させる
2. **検証を組み込む** — 「変更後にテストを走らせて結果を報告して」
3. **人間はジャッジに集中** — 実装はAI、方向性・承認は人間
4. **大タスクは段階分解** — 一度に詰め込まず、ステップで進める

## 7原則のClaude Code適用

| 原則 | Claude Codeでの実践 |
|---|---|
| ゴールを先に設定 | 達成したい機能・成果物を最初に明示 |
| 条件を全て先に出す | 制約・既存実装・規約をCLAUDE.mdと初回指示に |
| 段階的に進める | 計画→実装→テスト→PRを分けて進行 |
| 選択肢を比較させる | 「実装方針を2案出して比較して」 |
| 一度に詰め込みすぎない | サブタスクに分割、必要ならサブエージェント活用 |

## Claude Code固有のTips

- **CLAUDE.md = 文脈エンジンの本体**: まずここを整備するのが最優先
- **Skills / カスタムコマンド**: 繰り返すワークフローを定型化
- **MCP連携**: GitHub・DB・外部サービスをワークフローに組み込む
- **長時間タスクはOpus 4.8**: 最初の1ターンに全仕様を渡すと自律的に完遂しやすい（条件先出しの原則）
- **サブエージェント**: 並列・独立した作業はサブエージェントに委任（AIRFLOW的な担当割り当て）
