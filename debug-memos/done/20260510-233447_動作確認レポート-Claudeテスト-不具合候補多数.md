## 環境
- URL: https://shunjapanes.github.io/main/
- ブラウザ: Chrome (Mac)
- テスト方法: ファイル新規作成→TSVペースト/直入力→検索置換→ソート→フィルタ→シート追加→ヘルプ参照

═══ ★重大な不具合 ═══

### 【A】TSVペーストが1列分しか展開されない（致命的）
[手順]
  1. ファイル→新規（Sheet1.tsvは1列1行で開始）
  2. 「名前\tスコア\t地域\nたろう\t85\t東京...」をクリップボードに用意
  3. セル(1,1)選択 → Cmd+V
[実際]
  - 改行は認識され6行に展開される
  - しかし列方向は1列目のみが入り、2列目(スコア)/3列目(地域)が表示されない
  - 「データ→右に列挿入」で列を増やすと、隠れていた2/3列目のデータが現れる
[期待]
  - ペースト時に必要列数まで自動拡張、もしくは確認ダイアログ

### 【B】セル直入力でTabキーがセル間移動として機能せず、データ消失
[手順]
  1. 新規Sheet1.tsvでセル(1,1)選択
  2. 「名前」入力→Tab→「スコア」入力→Tab→「地域」入力→Enter→次行...
[実際]
  - Tabが想定通りのセル移動として機能せず、最初の列の値が保存されない
[期待]
  - Tab=次セル、Shift+Tab=前セル、Enter=下セル

### 【C】ソート時に1行目（ヘッダー候補行）も並べ替え対象になる
[手順]
  1. 1行目に見出し「名前/スコア/地域」、2行目以降にデータ
  2. ホーム→「昇順」または列ヘッダークリック
[実際]
  - 「名前」「スコア」「地域」もデータとして並べ替えられ末尾に移動
[期待]
  - 「1行目をヘッダーとして扱う」スイッチを追加

### 【D】Cmd+Z/Cmd+Shift+Z（Mac標準のUndo/Redo）が機能しない
[手順]
  1. 何かデータ操作（置換、編集等）
  2. グリッド上のセルをクリック→Cmd+Z
[実際]
  - グリッドのデータが戻らない
  - 検索ボックスにフォーカスがある時は検索文字列の文字が消えるだけ
  - ホームタブの「元に戻す」ボタン経由ではUndo成功
[期待]
  - Macで Cmd+Z/Cmd+Shift+Z を Ctrl+Z/Ctrl+Y のエイリアスとして処理

### 【E】「列統計」ボタンが完全に無反応（機能未実装の可能性）
[手順]
  1. 数値列のセルを選択 or 何もしない
  2. ツール→「列統計」をクリック
[実際]
  - ダイアログ・パネル・ステータスバー全て無反応
  - コンソールエラーも出ない
[期待]
  - 選択列の件数・合計・平均・最小・最大などをポップアップ表示

### 【F】★メモダイアログのフォーカス管理バグ（このメモ作成時に発覚）
[手順]
  1. ツール→「メモ」でデバッグメモを開く
  2. 内容欄に長文を入力
[実際]
  - クリックでフォーカスを当てたつもりが、背後のグリッドにフォーカスが移ることがある
  - 「メモ内容」のつもりで打った長文が裏のシートのセルへ流れ込み、シート134行目までデータが書き込まれた
[期待]
  - モーダル表示中はモーダル外のキー入力をブロック（フォーカストラップ）

### 【J】ステータスバー右下の位置情報が様々な操作後に更新されない

## 対応記録
- 対応日: 2026-05-10
- 対応内容:
  - 【A】doPasteOverwrite に renderHeader() を追加し、ペースト後に新列がヘッダーに反映されるよう修正
  - 【D】App.tsx に親フレームから undo/redo を転送するキーハンドラーを追加（検索欄フォーカス時でも Cmd+Z が動作）
  - 【E】showColStats の alert() をカスタムモーダルに変更（モバイルiframe内でのブロック解消）
  - 【F】onKeyDown にメモダイアログ開放時のガードを追加し、背後グリッドへのキー流入を防止
  - 【H】SearchBar の Enter キーでデバウンスをキャンセルして即検索→ナビゲート
  - 【I】replaceAll 後に execSearch() を呼び出し、検索件数を更新
