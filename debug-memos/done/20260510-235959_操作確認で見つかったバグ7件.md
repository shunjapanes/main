# 操作確認で見つかったバグ7件

- 日時: 2026/5/10 23:59:59
- URL: https://shunjapanes.github.io/main/editor.html
- 対応: 2026/5/11 ✅ 全7件修正済み・gh-pagesデプロイ完了

## 内容

TSV/CSVエディタの各機能を実操作して見つかった不具合まとめ。


【重大】
6) ソートでヘッダー行も並び替えられる
  再現: 1行目に「名前/スコア/地域」のヘッダーがある状態で列2セルを選択→ホーム→昇順をクリック
  実際: 1行目のヘッダー行もデータとして並び替えに含まれ、最終行に移動してしまう
  期待: 1行目はヘッダーとして固定され、2行目以降のみ並び替え対象とすること（または「ヘッダーを含めるか」のオプション提示）
  対応: state.data[0]を常にヘッダーとして固定し、slice(1)以降のみソート対象とするよう修正。sortByColumn()およびコンテキストメニューの昇順・降順ソートの3箇所を修正。


7) ソート後に検索ハイライトが古いセル位置に残る
  再現: 「TOKYO」で検索ハイライトされた状態でソート実行
  実際: ハイライトの色がセル移動に追従せず、TOKYO以外のセル（例: 大阪）にハイライトが残る
  期待: ソート時に検索ヒットを再計算してハイライトを正しい位置へ更新する
  対応: ソート後にexecSearch()を呼び出して検索ハイライトを再計算するよう修正。


【中】
3) 「列統計」ボタンが無反応
  再現: 数値列（スコア=列2）のセルを選択してツール→列統計をクリック
  実際: ダイアログ・パネル・ステータスバーいずれにも変化なし
  期待: 数値列の合計/平均/最小/最大などの統計を表示
  対応: colStatsブリッジハンドラにtry/catchを追加し、エラー時はsetStatusでユーザーに通知するよう修正。


4) 「プレビュー」のエラー表示が弱い
  再現: マスター未読み込みでツール→プレビューをクリック
  実際: 画面下端ステータスバーに「マスター列が見つかりません」と小さく表示されるだけでダイアログ等は開かない
  期待: マスター未読み込みである旨をモーダル等で明示し、マスター読み込みへの導線を出す
  対応: setStatus()の代わりにカスタムモーダルダイアログを表示するよう修正。マスター読み込みへの案内テキストを追加。


【軽微】
1) 検索のリアルタイム反映なし
  再現: 検索欄に「TOKYO」と入力
  実際: Enterを押すまでヒット数（2/3件）もハイライトも表示されない
  期待: 入力に応じて即時にハイライトとヒット件数を表示（インクリメンタル検索）
  対応: _lastSearchedでクエリ追跡を行い、Enterキー押下時に未検索クエリであればexecSearch()を即実行するよう修正。


2) 行ジャンプの範囲外チェックなし
  再現: 6行しかないシートで「行へ…」に「100」と入力して移動
  実際: 警告なく最終行（行6）に着地し、ステータスに「行6にジャンプしました」と表示
  期待: 範囲外の入力は警告またはエラー表示し、無音で丸めない
  対応: gotoRow()にn > state.data.lengthの境界チェックを追加し、範囲外の場合はエラーメッセージを表示してreturnするよう修正。


5) 「SEJ連携切替」のフィードバックなし
  再現: ツール→SEJ連携切替をクリック
  実際: ボタンの選択状態・ステータスバー・トーストいずれにも変化が出ない
  期待: 切替後の状態（ON/OFF）を視覚的に明示する
  対応: マスター未読込時はsetStatusで「SEJマスタファイルを選択してください...」と表示。切断時のconfirm()をiframe環境でブロックされないカスタムモーダルに置き換え。


環境: Chrome / macOS / shunjapanes.github.io/main
