# チケット料金表自動生成 (ticket_pricing_gas) Skill

## 概要

公式サイトのURLを入力するだけで、Gemini APIが全ページを自動巡回し、5シート構成のチケット料金表スプレッドシートを自動生成するGoogle Apps Scriptツールです。

**生成されるシート:**

| シート | 内容 |
|---|---|
| Sheet1: 料金表 | 券種 × 年齢区分 × 前売/当日価格 |
| Sheet2: 入場方法 | 電子チケット/プラスチックパス有無 |
| Sheet3: 利用エリア | 利用可能エリア・特典 |
| Sheet4: 決済・オプション | セブン発券/リセール/アップグレード可否 |
| Sheet5: アップグレードマトリクス | 券種間の変更可否表 |

**対応サーキット例:**
- 鈴鹿サーキット（SUPER FORMULA など）: `https://www.suzukacircuit.jp/superformula/1/`
- モビリティリゾートもてぎ（2&4 など）: `https://www.mr-motegi.jp/2and4/`

## セットアップ（初回のみ）

### STEP 1: Gemini APIキーの取得

1. Google AI Studio にアクセス
2. 「Create API Key」でキーを取得・コピー

### STEP 2: スプレッドシート作成

1. Google ドライブで新しいスプレッドシートを作成（名前例: 「チケット料金表生成ツール」）
2. 「拡張機能」→「Apps Script」を開く
3. ファイル構成（2ファイル）:
   - `コード.gs` → **config.js** の内容を貼り付け
   - `main.gs` → 「+」でファイル追加し、**main.js** の内容を貼り付け
4. 保存（Ctrl+S）

### STEP 3: APIキーの設定

1. Apps Script の左メニュー「⚙ プロジェクトの設定」をクリック
2. 「スクリプトプロパティ」→「スクリプトプロパティを追加」
3. プロパティ名: `GEMINI_API_KEY` / 値: STEP 1で取得したAPIキー
4. 「スクリプトプロパティを保存」

### STEP 4: 初期設定の実行

1. Apps Script エディターで関数 `setupConfig` を選択して ▶ 実行
2. 権限の承認を求められたら「許可」
3. スプレッドシートに戻り、CONFIGシートのB2セルにベースURLを入力

---

## 実行

**方法1: メニューから実行**
スプレッドシートのメニュー「🎫 チケット料金表」→「料金表を生成」

**方法2: Apps Scriptから実行**
関数 `main` を選択して ▶ 実行

**処理時間の目安:** 約2〜4分（券種数による）
**進捗表示:** CONFIGシートのB5セルに表示

---

## 処理フロー

```
1. /ticket/ ページ → 券種一覧を抽出
2. 各券種の個別ページを巡回 → 料金・条件を抽出
3. /ticket/about/ ページ → 手数料・リセール条件を抽出
4. 全データ統合 → 5シート分の構造化JSONを生成
5. アップグレード可否のマトリクス計算
6. スプレッドシートに書き込み
```

## 出力検証チェックリスト

生成後、必ず公式ページと以下の項目を照合してください。

### Sheet1（料金表）検証

| 確認項目 | 確認方法 |
|---|---|
| 全券種が網羅されているか | 公式の `/ticket/` ページと比較 |
| 年齢区分が正しいか | 各券種の詳細ページと比較 |
| 前売価格が正しいか | 数字を1件1件照合 |
| 当日価格が正しいか（当日販売ありの場合）| 当日販売なしの場合は88888表記 |
| 子ども・幼児区分の年齢定義が正しいか | 例：U23、中学生以下など |
| セット券・同時購入商品の表記 | 矢印（→）で連結商品を示す |

### Sheet2（入場方法）検証

| 確認項目 | 確認方法 |
|---|---|
| 電子チケット対応可否 | 公式ガイドページで確認 |
| プラスチックパス発行有無 | 公式ガイドページで確認 |
| 駐車場入場方法 | 駐車券の種類と入場手段 |

### Sheet3（利用エリア）検証

| 確認項目 | 確認方法 |
|---|---|
| 利用可能エリアの名称 | 座席図・エリア案内と比較 |
| 特典の記載 | 各券種の詳細ページで確認 |

### Sheet4（決済・オプション）検証

| 確認項目 | 確認方法 |
|---|---|
| セブン-イレブン発券手数料 | 約220円が標準 |
| リセール出品手数料（無料が標準） | 公式ガイドで確認 |
| リセール譲渡手数料 | 約220円が標準 |
| アップグレード可否・最低手数料 | 公式ガイドで確認 |
| 当日価格切り替え日時 | 正確な日時を確認 |

### Sheet5（マトリクス）検証

| 確認項目 | 確認方法 |
|---|---|
| 対象外券種が除外されているか | BOX席など特殊券種 |
| アップグレード方向の正確性 | 安い席→高い席のみ可が基本 |
| 年齢変更（U23→一般）対応可否 | 公式ガイドで確認 |

## よくあるエラーと対処法

| エラー | 原因 | 対処 |
|---|---|---|
| CONFIGシートが見つかりません | `setupConfig()` 未実行 | setupConfigを先に実行する |
| APIキー未設定 | スクリプトプロパティ未設定 | GEMINI_API_KEYを設定する |
| 時間制限で途中スキップ | 券種数が多い（6分制限） | 翌日再実行（GASのリセット後） |
| age_prices未検出 | ページ構造の変更 | 該当券種を手動入力で補完 |
| 特定券種の価格が空 | 個別ページのHTML構造が特殊 | 公式ページから手動で記入 |

---

## 注意事項

- **使用モデル:** gemini-2.5-flash（速度・コスト優先）
- **GASの実行時間制限は6分**。大量の券種がある場合はタイムアウトの可能性あり
- **APIキーはスクリプトプロパティに保存**（CONFIGシートには書かない）
- **生成後、必ず公式ページと突き合わせて内容を確認**してください
- HTMLの構造変更があった場合、抽出精度が低下する可能性があります
- 価格・販売条件は推測で断定しない。根拠がない場合は `要確認` とマークする

## 作業手順（毎回のレース対応）

### 1. 準備確認

実行前に確認:
- [ ] 公式サイトのチケット販売ページが公開済みであること
- [ ] CONFIGシートのB2にベースURLが設定されていること
- [ ] Gemini APIキーが有効であること

### 2. 実行

- [ ] `main` 関数を実行
- [ ] 進捗をB5セルで確認（約2〜4分）
- [ ] 完了ダイアログでエラー件数を確認

### 3. 出力検証

- [ ] Sheet1: 券種数・価格を公式ページと照合
- [ ] Sheet2: 入場方法を確認
- [ ] Sheet3: エリア情報を確認
- [ ] Sheet4: 手数料・当日価格切り替え日時を確認
- [ ] Sheet5: マトリクスの対象外券種を確認

### 4. 修正・補完

エラーや未取得データがある場合:
- 該当セルに手動で値を入力
- セルに `[要確認]` プレフィックスを付けて未確定箇所をマーク

### 5. 共有・活用

完成した料金表は以下の用途に活用:
- レース公式サイトの料金表ページ更新（`/race-site-update-reviewer` と連携）
- 広報・制作会社への料金情報提供
- 社内チケット販売業務の参照資料
