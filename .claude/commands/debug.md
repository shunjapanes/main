このプロジェクト（TSV/CSVエディター）のデバッグ作業を行います。**テスト実行**と**コードレビュー**の両方を実施してください。

## Step 1: Playwrightテストを実行

```bash
cd /home/user/main/client && PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers npx playwright test --timeout=30000 2>&1
```

テスト結果を確認し、失敗したテストがあれば原因を特定します。

## Step 2: 失敗テストの分析

失敗があった場合:
- エラーメッセージと行番号を確認
- 該当するソースコード（`client/public/editor.html` または `client/src/`）を読み込む
- バグの根本原因を特定する

## Step 3: 変更差分のコードレビュー

```bash
git diff main...HEAD -- client/public/editor.html client/src/
```

差分を確認して、以下の観点でレビューします：

- **バグの可能性**: 条件式の誤り、状態管理の抜け、非同期処理の競合
- **エッジケース**: 空データ・0件・境界値の扱い
- **副作用**: 他の機能への影響（特にfilter/view state/tab管理）
- **最近修正した箇所との整合性**: 新規ファイルロード時のビュー状態リセット等

## Step 4: 報告

以下の形式で結果をまとめます：

```
## テスト結果
- 合計: N件
- 成功: N件 ✓
- 失敗: N件 ✗ （失敗している場合は詳細）

## コードレビュー指摘
- [重要度: 高/中/低] 指摘内容
- ...

## 推奨アクション
- 修正が必要な場合は具体的な変更箇所を提示
```

失敗しているテストや問題のあるコードが見つかった場合は、修正するかどうか確認してから実施してください。
